# Code of Conduct

MechFerret is a research engineering project. We want discussion to stay
rigorous, practical, and welcoming to contributors with different backgrounds in
software, mechanistic interpretability, machine learning, security, and
scientific writing.

## Expected Behavior

- Be respectful and direct. Critique ideas, claims, code, artifacts, and
  evidence rather than people.
- Keep technical disagreements grounded in reproducible commands, files,
  examples, or benchmarks when possible.
- Assume public issues and pull requests may be read by new contributors.
  Explain context when it affects correctness or safety.
- Protect private data. Do not post secrets, private source material, provider
  logs, or sensitive run artifacts in public threads.
- Preserve the offline-first project contract unless an optional integration is
  explicitly in scope.

## Unacceptable Behavior

- Harassment, threats, insults, or discriminatory language.
- Publishing another person's private information or credentials.
- Deliberately derailing issue threads, reviews, or release work.
- Submitting misleading artifacts, forged verification output, or knowingly
  unsafe reproduction steps.

## Reporting

For conduct concerns, open a brief issue asking for maintainer contact or reach
out through the repository owner profile. For vulnerability reports, follow
the [Security Policy](SECURITY.md).

Maintainers may edit, hide, or close comments and issues that violate this
policy, and may block repeat offenders from the project.
